Convert Figma-exported SVG into Android's Adaptive Icon

Posted on December 19, 2023
Tags: Code

Lately, I decided to switch our Expo app to use adaptive icons on Android. The Android version used to share the icon with iOS (by assigning expo.android.icon the same value as expo.icon in app.json). However, this looked terrible on Android systems where circular launcher icons are used. Adopting to adaptive icons will make the app not only more polished, but also integrate better with the latest Android‚Äôs visual effects and user theming.

The Challenges

The icon was designed on Figma but not based on any Adaptive Icon Template, so it takes some effort to convert it into a usable format for Android.

The challenges are:

  • Some parts of the icon are made of clipping masks 1 in Figma, as such it is not straightforward to extract the foreground contours as closed curves.
  • For whatever reason, the exported SVG is not rendered correctly in Inkscape.
  • Additionally, I only have access to read-only mode in Figma.

Icon Size and Safe Zone

First of all, how to choose the icon size? According to Android adaptive icon guidelines, it is an icon in a 72x72 dp safe zone in a 108x108 dp container. For native Android development, using a 108x108 SVG should be preferred, but AFAIK Expo doesn‚Äôt support SVG format for Android adaptive icons. react-native-svg-app-icon looks promising, but it seems more suitable for bare React Native rather than Expo.

So, if we have to deal with raster images, what about the resolution? Expo‚Äôs documentation suggests at least 512x512, and I think 1024x1024 should be a good choice as it is the same as the iOS one.

Expo‚Äôs default icon (located at ./assets/images/adaptive-icon.png and is also 1024x1024) has some nice construction lines on it to help designers align the foreground content, which should just fit within the outermost circle.

From Designing Adaptive Icons (It is a blog run by Google Design so I assume it is semi-official),

Adaptive icons are 108dp*108dp in size but are masked to a maximum of 72dp*72dp. Different devices can supply different masks which must be convex in shape and may reach a minimum of 33dp from the center in places.

Because of the minimum reach of the mask, you can consider a centered 66dp diameter circle as a safe zone, guaranteed not to be clipped.

The Solution

From the analysis above, just fit the main element of the icon inside the circle of safe area, and rasterize the SVG to 1024x1024 image should produce an acceptable result.

Step-by-step:

  • In Figma‚Äôs ‚ÄúLayers‚Äù panel in the left sidebar, right click the entity representing the app icon, and copy it as SVG.

  • Use free services like SVGViewer to prettify the SVG markup.

  • Now, take a look at the SVG code. It is on a 327x327 canvas and has a circle of radius 163.5 as a mask:

    <svg width="327" height="327" viewBox="0 0 327 327" fill="none" xmlns="http://www.w3.org/2000/svg">
  <mask id="mask0_3983_11989" style="mask-type:alpha" maskUnits="userSpaceOnUse" x="0" y="0" width="327" height="327">
    <circle cx="163.5" cy="163.5" r="163.5" fill="#FF612F"/>
    <circle cx="163.5" cy="163.5" r="163.5" fill="url(#paint0_radial_3983_11989)" fill-opacity="0.6"/>
  </mask>
  ...
</svg>

    As said, the outer circle in Expo‚Äôs example icon corresponds to the 66dp safe area. In raster graphics editor, I measure the diameter of the outer circle to be approx. 612 - 616px. Coincidentally, 616 gives a simpler fraction (616/66 = 28/3 v.s. 612/66 = 102/11). Anyway the difference is insignificant ‚Äì just some measurement error.

  • By 616 / 327 = 1.88379204892966, I apply a scale transformation to the graph:

    <svg width="1024" height="1024" viewBox="0 0 1024 1024" fill="none" xmlns="http://www.w3.org/2000/svg">
<g transform="scale(1.88379204892966) translate(103 103)">
  ...
</g>
</svg>

    It looks good in SVGViewer‚Äôs preview panel.

  • By deleting those foreground nodes from XML tree, I obtain the background icon ‚Äì it‚Äôs just a solid square with gradient fill.

  • Finally, use the method from Convert SVG to PNG using your browser to export both foreground and background as PNG images.

This may not be an ideal workflow, but it works.
