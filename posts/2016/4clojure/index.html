通关4Clojure

Posted on March 6, 2016
Tags: Code

这段时间终于做完了4Clojure上的全部练习题，代码在这里。 4Clojure是一个基于Koans（“公案”，原指禅宗里帮助开悟的教学片断）风格教学法的Clojure学习网站，挑战者需要完成一系列填空题，使给出的语句成立。感觉自己动手做练习确实比之前看《Clojure Programming》的印象来得深刻。说是通关，是因为这个教程确实像打游戏的关卡设计一样，算是编程学习游戏化（Gamification）的一个尝试。 156道题从最基本的语法题到来自Euler Project的难题，一级级循序渐进，就像升级打怪一样让你不知不觉就掌握了Clojure的语法和习语。

说缺点的话，主要是测试过于宽松。首先，数据量太小，即使存在O(n·lg n)时间复杂度的算法，写出O(n^2)甚至O(n^3)的代码一般也不会超时。很多时候也缺少边界情况的测试。其次，一些题目允许投机取巧的解法，比如第125题Gus’ Quinundrum，测试是 (= (str '__) (__))，我看到有人这么写：(fn [x] (str x x)) '(fn [x] (str x x))，并不是真正的quine但也能过关。

就像有些硬核游戏玩家不满足于通关还追求速通一样，4Clojure有一个参加code golf的选项，比赛谁提交的答案最短。规则是计算非空白字符。不过我觉得如果根据Lisp的特点，不按字符数而是按atom数计分可能更好，既能golf又保持代码可读性，也不用去折腾《短码之美》里面那一类语法的细枝末节。按本帖提示，想提高高尔夫水平的话可以在4Clojure上关注maximental，学习精简代码有关的奇技淫巧。
